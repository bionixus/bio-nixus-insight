01 — Purpose & Scope

Why every document needs a premium HTML version

Client expectations, brand consistency, and review speed all improve when external deliverables ship in a print-ready, premium HTML format.

This SOP applies to:

  • Client proposals and credentials decks
  • Case studies and portfolio appendices
  • Briefing decks and one-pagers
  • External newsletters and announcements

Design system: Print-first A4 documents use the BioNixus premium HTML conventions (EB Garamond + Barlow, navy/blue/gold palette). Do not mix website tokens into print deliverables without CMO approval.

02 — Document Checklist

Quality gate before HTML review

Use this checklist for proposals, credentials, case studies, newsletters, and briefing decks.

A. Draft content (owner / subject matter expert)

  • Core message, value proposition, and key claims finalized
  • Facts, figures, and product names verified against source documents
  • Sensitive pricing marked [TBD — CFO input] until confirmed
  • No fabricated client logos or testimonials

B. HTML conversion (creative / production)

  • Clone nearest approved template from the premium HTML template library
  • Apply brand tokens: navy #002244, blue #0069A3, gold #B8862D
  • Use exactly one <h1> on the cover; interior sections use <h2 class="section-title">
  • Replace all {{PLACEHOLDER}} tokens with real copy or mark as [TBD]
  • Relative asset paths for logos, stamps, and signatures
  • Descriptive alt text on every image
  • Naming follows: bionixus-{client-slug}-{topic-slug}-{type}-{year}.html

C. Quality & compliance review

  • Fonts load in preview (Google Fonts import present)
  • Print CSS intact: backgrounds print, page breaks avoid tables and signatures
  • WCAG AA contrast verified on text/background combinations
  • No invented pricing or confidential client data exposed
  • Legal / CEO approval recorded on parent issue before external send
03 — Workflow & Handoff

From draft to client delivery

A clear ownership chain keeps quality high and turnaround fast.

01 Draft Subject matter expert writes final content; validates facts and claims.
02 HTML Conversion Creative converts to branded premium HTML using the approved template and design tokens.
03 Review Legal / CEO approves content before any external distribution.

Handoff artifacts: Folder with HTML file + referenced brand assets + template diff summary. Post both on the task issue and move status to in_review.

Handoff Point Inputs Owner Outputs
Draft → HTML Finalized copy + approved assets + template file Creative Art Director Premium HTML file in deliverables/html/
HTML → Legal / CEO HTML preview + asset bundle + checklist CMO / CEO Assistant Approval record on task issue
Approved → Client Signed-off HTML + PDF export Client Partner / CEO Delivery confirmation
04 — Brand & Production Standards

Non-negotiable rules of the road

Filename: bionixus-{client-slug}-{topic-slug}-{type}-{year}.html — lowercase, hyphenated, no spaces or underscores.

Types:

  • proposal — client pitch / research proposal
  • credentials — company/partner capabilities
  • portfolio — selected case-study visual summary
  • briefing — executive one-pager or short deck

Title tag: BioNixus — {Topic} | {Client} {Year}

Print export: Chrome or Edge → Print → Save as PDF with "Background graphics" enabled. Use high-res stamps from brand assets when required.

Escalation: If a deliverable blocks on non-standard content or missing assets, raise to CEO and note the exception on the task issue.
