# PARA Knowledge Graph Index

This is the root of your PARA (Projects, Areas, Resources, Archives) knowledge graph.

## Directory Structure

- **projects/** - Active work with clear goals and deadlines
- **areas/** - Ongoing responsibilities with no end date
  - **people/** - Individuals in your professional/personal network
  - **companies/** - Organizations you interact with
- **resources/** - Reference material and topics of interest
- **archives/** - Inactive items from any of the above categories

## Quick Start

### Creating a New Entity

1. Determine the category (project/area/resource)
2. Create a directory: `mkdir -p {category}/{name}`
3. Create `summary.md` for quick context
4. Create `items.yaml` for atomic facts
5. Add entry here in the appropriate section below

### Entity Lifecycle

- **Projects** start active, move to archives when complete
- **Areas** remain active indefinitely
- **Resources** grow as you learn
- **Archives** preserve history without cluttering active lists
